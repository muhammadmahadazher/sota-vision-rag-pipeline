🎯 **What:** Replaced a raw exception string with a generic error message in `backend/app/api/stream.py` during WebSocket communication.
⚠️ **Risk:** Sending raw exception strings can expose sensitive system information, internal paths, or database details to connected clients, potentially leading to information disclosure vulnerabilities.
🛡️ **Solution:** The vulnerability is fixed by displaying a generic user-friendly message (`"An internal error occurred while processing the frame."`) to the client, while continuing to securely log the actual exception internally on the server using `logger.error`.
