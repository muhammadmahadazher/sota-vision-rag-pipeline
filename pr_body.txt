🎯 **What:** Inserted an explicit `except WebSocketDisconnect:` block inside the websocket stream's `receive_bytes` loop.
💡 **Why:** A broad `except Exception:` block was swallowing the `WebSocketDisconnect` signal raised by the client dropping the connection. This explicit check allows the disconnect to bubble up naturally, improving code maintainability and ensuring the framework accurately detects the disconnect without raising unnecessary errors.
✅ **Verification:** Ran `python -m unittest discover` successfully. Evaluated the fix logically and verified there are no broad impacts since only a correctly identified specific exception skips the generic trap.
✨ **Result:** Better control and handling over websocket disconnects in `process_frames_consumer`.
